# Analysis Prompt Candidate 2: "Persona Extraction"

**Prerequisites:** Read and follow all rules in `shared-rules.md` before proceeding.

Strategy: Top-down. Infer the creator's persona, goals, and audience first, then derive design rules as consequences of those higher-level decisions. Focuses on *why* choices were made.

---

## Your Approach

### Phase 1: Audience & Intent
- Who is this designed for?
- What is the creator trying to achieve?
- What context is this presented in?
- What's the power dynamic between creator and audience?

### Phase 2: Communication Strategy
- Argument structure: deductive or inductive?
- Persuasion model: logos / pathos / ethos ratio?
- Information pacing: front-loaded or suspense-built?

### Phase 3: Design Choices as Communication
For every major visual choice (font, color, density, layout), explain WHY — what it communicates to the audience. Express as general principles, not slide descriptions.

### Phase 4: Decision Framework
Express the creator's design process as 15-25 general IF-THEN-BECAUSE rules:

```
IF [content situation] THEN [design choice] BECAUSE [communication goal]
```

Rules must be GENERAL — work for any topic. Cover:
- When to create a new slide vs add to existing
- When to use text vs visual
- When to use data vs narrative
- How to open and close
- How to handle complexity
- How to create emphasis

### Phase 5: Anti-Patterns
General prohibitions — things this style never does.

## Output Format

```
# Design Persona: [Name this archetype]

## Core Identity
- **Optimizing for**: ...
- **Audience model**: ...
- **Communication mode**: ...

## Visual System
### Typography — [font, size, weight, color, and WHY for each role]
### Color Palette — [color, role, emotional function]
### Layout Templates — [name, structure, use-when trigger]
### Spacing & Density — [whitespace target, max elements, text-to-visual ratio]

## Decision Framework
[15-25 IF-THEN-BECAUSE rules — all general]

## Sequencing Architecture
[Opening/section/transition/closing formulas]

## Anti-Patterns
[General prohibitions]
```

**Remember: no slide numbers, no content summaries, no deck-specific details. General rules only.**
